Canvas och WebGL

Två stora nya saker i HTML5 standarden är Canvas och WebGL.
Canvas är ett api för att rita 2D raster grafik, många webbläsare idag har stöd för hårdvaru accellererat canvas vilket låter lite mer krävande 2D spel att köra via Canvas. WebGL är ett API för 3D grafik genom en lättare variant av OpenGL som är en av de största standarderna för 3D grafik.







Ljud och Bild

Såsom det finns flera bildformat för bilder finns det flera video och ljudformat för filmer och musik. För att kombinera olika format under en fil finns container filer. Ett vanligt container format är avi, avi filer kan inehålla både video, ljud och undertexter. För att kunna läsa och skriva till alla dessa format i container filerna finns konverterare som kallas för kodek. Därför kan man inte säga att en videospelare klarar av att spela up avi filer.

Det vanligaste video formatet idag kallas för H.264. Formatet används ofta för höguplöst video just av den anledningen att det komprimerar video väldigt bra. H.264 använder vanlig bild för bild komprimering. Istället sparar man bara skilnaden från den föregående bilden. Några nackdelar:

  • Dom kräver mycket kraft för att kodas/avkodas
  • Filmer som ändras ofta tar mer plats
  • Om det är fel någonstans i filmen påverkas ofta flera bilder efter felet

För att koda och avkoda hög uplöst video snabbare används ofta olika Grafik acceleratorer såsom grafikkort. Den tekniken kallas för hårdvaru-accelereation.

Vid applicationer som inte tillåter sökning i datan finns filformatet Ogg, Ogg än nämligen uppbygt så datan kän läsas i sekvens utan att läsaren behöver information längre fram. Det kan vara väldigt praktist i tex websidor eller spel, dvs då hela ljudfilen inte behövs på en gång. Ogg är ochså ett bra exempel på ett container format som bara stödjer ett fåtal kodek.
